RunBuggy Repo-Site Scoring Project

This project tackled the challenge of repo-site transparency on the RunBuggy car-shipping platform, where inconsistent site quality undermines customer and lender confidence. Our goal was to transform disparate internal order data and unstructured feedback into clear, data-driven “good vs. bad” repo-site scores that both consumers and RunBuggy teams can trust.

Data Integration & Preparation

We merged RunBuggy’s internal order records with public business data (Google, Yelp) using Python and SQL, cleaning and standardizing fields to create a unified dataset. Unstructured customer comments were processed with LLMs and custom Python scripts to extract numeric sentiment and reliability metrics.

Unsupervised Modeling

K-Means Clustering (scikit-learn)

  • Grouped 500 repo sites into four discrete quality tiers based on total ratings, cancellation rates, and average Google score.

Principal Component Analysis (PCA)

  • Reduced high-dimensional feature space (Google/Yelp ratings & counts, message-response rate) to a single continuous “customer-satisfaction” score, capturing nuance within clusters.

Results & Impact

The clustering model yielded clear categorical tiers that align with user expectations (e.g., “high-trust” vs. “risk” sites), while the PCA score highlighted fine-grained differences across all locations. These scores powered a prototype interactive map, enabling customers to filter sites by quality and confidence level. Internally, RunBuggy teams gained a robust scoring pipeline—complete with automated text-to-numeric preprocessing—that will guide site vetting, improve lender relations, and enhance user satisfaction.
